# 경기도 응급의료 자원 분포 대시보드

경기도 응급의료기관의 **지역별 분포와 형평성**을 한눈에 보여주어 **응급의료 사각지대**(자원 부족 지역)를 드러내는 대시보드입니다.

**🔗 라이브 데모**: https://anywaysoul92.github.io/gyeonggi-emergency-dashboard/

---

## 이 대시보드는 무엇인가요?
경기도 전역의 응급의료기관 72곳을 시군별 분포와 인구 대비 형평성 관점에서 보여주는 대시보드입니다. "응급실 뺑뺑이"의 구조적 원인인 지역 간 자원 격차와 사각지대를 드러내는 것이 목적입니다.

## 누가, 언제 보나요?
경기도·시군의 응급의료 정책 담당자와 응급의료지원센터 기획자가 주 사용자입니다. 분기·반기 정책 검토나 예산 편성, 신규 기관 지정을 논의할 때 사각지대 식별과 근거 자료로 활용합니다.

## 어떤 지표(KPI)를 보여주나요?
상단에 네 가지 지표를 카드로 고정 배치합니다.

| KPI | 값 | 의미 |
|---|---|---|
| 총 응급의료기관 | 72개 | 경기도 전체 자원 규모 |
| 권역응급의료센터 | 9개 | 중증대응 최상위 등급 |
| 응급의료 사각지대 | 6개 시군 | 기관 0곳 (본 데이터 기준) |
| 인구 10만명당 평균 | 0.53개 | 형평성 기준선 |

## 어떤 차트가 있나요?
세 가지 차트로 구성했습니다.

- **시군별 인구 10만명당 기관 수** (가로 막대): 인구 대비 부족 지역이 아래쪽에 드러남. 기관 0곳은 빨강.
- **등급 구성** (도넛): 권역 9 / 지역센터 33 / 지역기관 30.
- **이중 취약도** (산점도): 구조 취약(x=인구10만명당) × 규모(y=총 기관수).
- **실시간 응급실 가용병상** (표, 2단계): 응급의료정보제공(E-Gen) API에서 10분 주기로 동기화한 경기도 72개 병원의 가용병상·수술실·일반병상 현황.

## 핵심 인사이트는 무엇인가요?
경기도 31개 시군 중 6곳(하남·과천·의왕·가평·동두천·양주)에는 응급의료기관이 한 곳도 없고, 인구 10만명당 기관 수는 지역 간 약 10배(0.23 → 2.38)까지 벌어집니다.

## 데이터의 한계는 무엇인가요?
- 실시간 응급실 가용병상은 병원이 직접 입력한 시각(hvidate) 기준이라 완전한 실시간이 아니며, 입력이 늦어지면 실제와 차이가 날 수 있습니다.
- 시군 사각지대는 행정 지표이며 인접 시군 접근성과 다를 수 있습니다.
- 인구가 적은 시군은 10만명당 수치가 과대평가될 수 있어 절대 기관 수와 함께 봐야 합니다.
- 인구는 추정치이며, 공식 주민등록인구로 교체 시 정확도가 올라갑니다.

## 어떻게 보나요?
- 배포본: 위 라이브 데모 링크(GitHub Pages).
- 로컬: `index.html`을 브라우저로 열면 동일하게 표시됩니다.

## 데이터 출처와 관련 문서는?
- 데이터: 공공데이터 "경기도 응급의료기관현황"(시군명·병원명·업무구분명·주소·위경도), 경기도 시군별 인구(행정안전부 주민등록인구 기반 추정치).
- **데이터베이스**: Supabase(Postgres)에 `emergency_facilities`, `sigun_population`, `realtime_bed_availability` 테이블로 적재되어 있으며, 대시보드는 페이지를 열 때마다 이 DB를 읽어 KPI·차트를 계산합니다. 테이블에 데이터를 추가/수정하면 새로고침 시 바로 반영됩니다(별도 재배포 불필요). 읽기 전용 공개 키(anon key)만 사용하며 RLS로 조회만 허용되어 있습니다.
- **실시간 동기화**: Supabase Edge Function(`sync-realtime-beds`)이 공공데이터포털 응급의료정보제공(E-Gen) API를 호출해 `realtime_bed_availability`를 채웁니다. pg_cron이 10분마다 이 함수를 호출합니다. API 서비스키는 Supabase Secrets에만 저장되어 있고 프론트엔드 코드에는 포함되지 않습니다.
