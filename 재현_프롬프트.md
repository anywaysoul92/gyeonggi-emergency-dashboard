# 재현 가이드 — 경기도 응급의료 자원 분포 대시보드

이 문서만 보고도 동일한 대시보드(HTML)를 재현할 수 있도록 정리한 프롬프트/사양입니다.

## 0. 결과물
- 단일 HTML 파일(`응급의료_대시보드.html`). 브라우저에서 바로 열리며, 차트 3종 + KPI 4종을 표시.
- 차트 라이브러리: Plotly (CDN), 폰트: Pretendard (CDN).
- 디자인: Wanted "Montage" 디자인 시스템 토큰을 CSS로 재현.

## 1. 목표 / 맥락
- 주제: 경기도 응급의료 자원의 **지역별 분포·형평성**을 한눈에 보여 **사각지대**(자원 부족 지역)를 드러낸다.
- 배경: "응급실 뺑뺑이"의 구조적 원인(지역 간 자원 격차)을 진단하기 위한 정책·기획용 대시보드.
- 범위: 1단계는 정적 데이터 기반 구조 분석. (실시간 수용가능 정보는 별도 API가 필요한 2단계)

## 2. 데이터
- 원천 1: `응급의료기관현황.csv` (경기도 응급의료기관 및 응급의료지원센터 현황)
  - 컬럼: 시군명, 병원명/센터명, 업무구분명(등급), 대표전화번호, 도로명/지번주소, 우편번호, 위도, 경도
  - 인코딩: CP949(euc-kr). 유효 기관 72개.
  - 등급 구성: 권역응급의료센터 9 / 지역응급의료센터 33 / 지역응급의료기관 30
- 원천 2: `경기도_시군_인구.csv` (시군명, 인구) — 형평성 지표(인구 10만명당) 계산용. 행정안전부 주민등록인구 기반(추정치, 공식값으로 교체 가능).
- 집계 결과(핵심):
  - 커버 시군 25 / 경기도 전체 31 → **사각지대 6곳: 하남시·과천시·의왕시·가평군·동두천시·양주시**
  - 인구 10만명당 평균 0.53개

## 3. 디자인 시스템 (Montage 토큰)
GitHub `wanteddev/montage-web` 에서 추출한 값:
- Primary `#0066FF`, Primary-strong `#005EEB`, Primary-soft `#9EC5FF`
- 텍스트 `#171719`, 보조텍스트 `#5A5C63`, 대체배경 `#F7F7F8`, 구분선 `#E1E2E4`
- 상태색: 성공 `#00BF40`, 주의 `#FF9200`, 위험 `#FF4242`(strong `#E52222`)
- 폰트: "Wanted Sans" 계열, 웹은 Pretendard로 대체. 간격 스케일 8px 기반, 카드 radius 16px.

## 4. KPI & 차트 정의
- 상단 KPI 카드 4개: ① 총 기관 72 ② 권역센터 9 ③ 사각지대 6개 시군 ④ 인구 10만명당 평균 0.53
- 차트 1 (형평성): 시군별 **인구 10만명당 기관 수** 가로 막대(오름차순). 기관 0곳 시군은 빨강. y축 automargin으로 시군명 전체 표시, 좌우/하단 여백을 넉넉히 줘 값·축 눈금이 잘리지 않게. **평균선은 넣지 않음**(평균은 KPI 카드로 표기해 중복·겹침 방지).
- 차트 2 (등급 구성): 도넛. **서로 구분되는 3색**(권역=파랑 `#0066FF`, 지역센터=주황 `#FF9200`, 지역기관=청록 `#00BDDE`) + 조각 사이 흰 경계선. 이름은 하단 가로 범례, 조각엔 값만 흰 글씨로 가로 표기.
- 차트 3 (이중 취약도): 산점도(전체 폭). x=인구10만명당 기관수, y=총 기관수. 좌하단=취약. 사각지대는 빨강으로 분리 표시. **좌표가 거의 겹치는 밀집 도시는 상시 라벨을 생략하고 hover(커서)로 확인**, 정확한 순위는 막대그래프 참조.
- 하단: 사각지대 6곳 강조 패널 + 데이터 한계 접이식 안내.

## 5. 재현 프롬프트 (순서대로 입력)
1. "경기도 응급의료기관현황 CSV(시군명·업무구분명·위경도 포함, CP949)를 읽어서 시군별 기관 수, 등급 구성, 커버 시군 수를 집계해줘."
2. "경기도 31개 시군 기준으로 기관이 0곳인 사각지대 시군을 찾아줘. 시군별 인구(별도 CSV)를 결합해 인구 10만명당 기관 수를 계산해줘."
3. "Wanted Montage 디자인 시스템(색상 #0066FF 계열, Pretendard 폰트, 16px 카드)을 적용해서, KPI 카드 4개(총 기관·권역센터·사각지대·10만명당 평균)와 차트 3개(형평성 가로막대·등급 도넛·이중취약 산점도)를 담은 단일 HTML 대시보드를 만들어줘. 차트는 Plotly CDN 사용."
4. "막대는 시군명이 잘리지 않게 automargin, 도넛은 이름을 하단 범례로, 산점도는 라벨 겹침을 줄이고 사각지대는 빨강으로 분리해줘."
5. "데이터 한계(실시간 정보 없음, 행정경계 집계 한계, 소규모 시군의 10만명당 과대평가, 인구 추정치)를 접이식 안내로 하단에 넣어줘."
6. (다듬기) "막대 오른쪽 값이 잘리지 않게 x축 범위와 오른쪽 여백을 늘리고, 아래 축 눈금이 잘리지 않게 하단 여백도 확보해줘. 평균 점선은 숫자와 겹치니 제거하고 평균은 KPI 카드로만 표기해줘."
7. (다듬기) "도넛 색이 파란색 계열이라 구분이 안 되니, 서로 뚜렷이 구분되는 3색(파랑·주황·청록)과 흰 경계선으로 바꿔줘."
8. (다듬기) "산점도에서 좌표가 거의 같아 겹치는 밀집 도시는 상시 라벨을 빼고 hover로 보게 하고, 잘 떨어진 도시만 이름을 표시해줘."

## 6. 기술 스택 & 실행
- HTML 버전: `응급의료_대시보드.html` 더블클릭 → 브라우저에서 열림(인터넷 필요: CDN).
- Streamlit 버전(선택): `pip install -r requirements.txt` → `streamlit run app.py`.

## 7. 데이터 한계
- 실시간 수용가능·병상 정보 없음(정적 명부) → 구조적 분석에 한정.
- 기관 규모 미반영(대형/소형 동일 카운트, 등급으로만 보정).
- 시군 사각지대는 행정 지표이며 실제 이송 접근성과 다를 수 있음.
- 인구 적은 시군은 인구 10만명당 수치가 과대평가될 수 있음 → 절대 수와 함께 볼 것.

## 8. (선택) 2단계 실시간 확장
- 국립중앙의료원 응급의료정보조회서비스 OpenAPI(`ErmctInfoInqireService`)의 `getEmrrmRltmUsefulSckbdInfoInqire`(실시간 가용병상), `getSrsillDissAceptncPosblInfoInqire`(중증질환 수용가능)를 결합.
- 형평성(구조) × 실시간(수용여력)을 겹친 "이중 취약도"로 확장. 서비스 인증키는 비밀값으로 별도 관리(코드/문서에 미기재).
